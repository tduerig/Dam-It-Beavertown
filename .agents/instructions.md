# Antigravity AI Assistant Guidelines

## 1. Gameplay Rigidity over Perceived Performance
NEVER alter or destroy gameplay features, aesthetic durations, or core user experiences simply for "performance reasons". If a visual or interactive feature (like slow-melting terrestrial leaves taking 26 seconds to decay) is causing a mathematical bottleneck, you **MUST** seek mathematical optimizations beneath the hood (such as caching, matrix throttling, or geometry pre-computations) rather than truncating the gameplay impact. 

If no mathematical optimizations exist, you **MUST** document this in a proposed implementation plan and request explicit user authorization before simplifying the visual/gameplay fidelity dynamically. Do not silently degrade the user experience.

## 2. Neutral Science and Healthy Scepticism
Prompt agents to be neutral scientific observers.  For example when using a browser agent leave instructions open to include unexpected things not just your targeted behavior.  Don't over-specify what you'd like them to observe or find, because it doesn't count as a real observation if you unblinded the experiment (eg: rather than asking: "is the river on the left flowing more?" ask "Describe what you see, comparing flow rate between any rivers you see." and never ask "Are there more blue pixels on the right side of the screen?" and try to infer semantics from that.   Similar with play-testing, if you have to explain much beyond goals, you're not really play testing. The worst-case of that would be instructing our play-tester how to work around a bug.  Never do that, it doesn't have value.   Prefer unittests, integration tests vs expensive artificially constrained agentic play/performance/simulation-testing flows.   Generally, when you think something is working, question that assumption and make sure you're not seeing what you want to see and assuming all is well rather neutrally observing evidence of that the underlying assumption is true.
